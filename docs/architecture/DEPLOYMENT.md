# Render Deployment Guide & Checklist

This document serves as the implementation guide and checklist for deploying BookBounty on Render using a Render Blueprint (`render.yaml`), specifically targeting the **Free Tier** resources.

## 1. Architecture Overview
- **Backend:** Django 6.x REST API, deployed as a Render **Web Service** (Python).
- **Frontend:** React 19 + Vite, deployed as a Render **Static Site** (Node.js).
- **Database:** Render Managed **PostgreSQL** database.

## 2. Pre-Deployment Configuration Checklist
The following configurations have been implemented in the codebase to support Render's infrastructure.

### Backend
- [x] **`dj-database-url`**: Configured in `core/settings.py` to parse Render's internal `DATABASE_URL` for PostgreSQL connections, while smoothly falling back to SQLite for local development.
- [x] **`gunicorn`**: Included as the production WSGI HTTP server (`uv run gunicorn core.wsgi`).
- [x] **`whitenoise`**: Configured in `core/settings.py` to serve Django static files (such as the admin panel styles) directly from the gunicorn process.
- [x] **`psycopg`**: Included in dependencies to support PostgreSQL connections.
- [x] **Build Script (`build.sh`)**: A bash script that safely unsets `VIRTUAL_ENV` (a Render-specific Python quirk), synchronizes `uv` dependencies, runs migrations, collects static files, and creates the admin superuser using environment variables.

### Frontend
- [x] **Build Command**: `npm install && npm run build` correctly outputs to the `dist/` directory.
- [x] **Publish Directory**: The `render.yaml` specifies `staticPublishPath: dist`.
- [x] **API Base URL**: `VITE_API_BASE_URL` is configured to point to the deployed backend Web Service URL (`https://bookbounty-api.onrender.com/api`).
- [x] **Client-Side Routing**: Render's Static Site handles client-side routing automatically by serving `index.html` for unknown paths.

## 3. Blueprint Configuration (`render.yaml`)
The `render.yaml` file defines our Infrastructure as Code (IaC). It has been configured to ensure that only free resources are provisioned.
- [x] Specifies `plan: free` for the API Web Service.
- [x] Specifies `plan: free` for the PostgreSQL database.
- [x] Specifies `type: web`, `runtime: static` for the Frontend UI (static sites are free by default).
- [x] Maps `DATABASE_URL` from the PostgreSQL instance securely to the Django API service.
- [x] Pre-fills `VITE_API_BASE_URL` from the Django API service to the Frontend UI via environment variables.
- [x] Auto-generates a secure `SECRET_KEY` and `DJANGO_SUPERUSER_PASSWORD` on the first deploy.

## 4. Manual Dashboard Steps (Post-Deployment)
Once the Blueprint is synced and applied in your Render dashboard, you **MUST** manually populate the following environment variables for the **Backend Web Service**. These are explicitly excluded from the `render.yaml` for security:

- [ ] `GEMINI_API_KEY`: Your Google Gemini API key. *(Required for the AI Triage engine)*
- [ ] `OPEN_LIBRARY_CONTACT`: Your email address for the Open Library API. *(Required)*
- [ ] `EBAY_CLIENT_ID` *(Optional)*: For real market valuation instead of Demo Mode.
- [ ] `EBAY_CLIENT_SECRET` *(Optional)*: For real market valuation instead of Demo Mode.

*(Note: If `DEMO_MODE=True` remains set in the Render environment variables, the system will use mocked market data.)*

## 5. Accessing the Application
Once the deployment completes successfully:
- The frontend application will be available at: `https://bookbounty-ui.onrender.com`
- The API backend will be available at: `https://bookbounty-api.onrender.com`
- The Django admin panel can be accessed at: `https://bookbounty-api.onrender.com/admin` (You will log in using `admin` and the password automatically generated by Render during the initial build).
